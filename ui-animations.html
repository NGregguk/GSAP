Setup notes

CDN & imports

  • ES module CDN (Skypack) in js/gsapSetup.js; serve over HTTP(s).
  • Explicit plugin registration kept in one place.

Modules & structure

  • Scripts use type="module"; patterns are split into modules.
  • Controls here are demo-only; wire timelines to your app state in production.

Motion defaults

  • Transforms/opacity only; prefers-reduced-motion handled via gsap.matchMedia().

Pattern 1

Hero Parallax Focus

Concept: Move layered elements at different speeds to create depth tied to input.

Live demo

Interaction

Guide users with depth.

Foreground and accent layers respond to input; tap Reset to return to neutral.

Implementation Notes

Core pattern

Create a lightweight parallax state driven by pointer input, then reset to neutral when you leave or tap reset.

View JS for this pattern 
const hero = document.querySelector(".hero");
const circle = hero.querySelector(".hero__circle");
const content = hero.querySelector(".hero__content");
const bg = hero.querySelector(".hero__bg");

const handleMove = (e) => {
  const rect = hero.getBoundingClientRect();
  const x = ((e.clientX ?? rect.width/2) - rect.left) / rect.width - 0.5;
  const y = ((e.clientY ?? rect.height/2) - rect.top) / rect.height - 0.5;
  gsap.to(circle, { x: x * 30, y: y * 30, duration: 0.3, ease: "power2.out" });
  gsap.to(content,{ x: x * 12, y: y * 10, duration: 0.3, ease: "power2.out" });
  gsap.to(bg,     { x: x * 8,  y: y * 6,  duration: 0.35, ease: "power2.out" });
};

hero.addEventListener("pointermove", handleMove);
hero.addEventListener("pointerleave", () => gsap.to([circle,content,bg], { x:0, y:0, duration:0.4 }));
How to apply
  • Markup: keep three layers inside .hero (.hero__bg, .hero__circle, .hero__content).
  • Import initHero (bundled via ui.js) and call after DOM ready; no ScrollTrigger needed.
  • Add a reset button if you want manual neutral state; otherwise pointerleave handles it.
  • Serve over HTTP(S) so the ES module imports resolve correctly.
  • Example page: Hero parallax layout
What to tweak
  • Parallax strength: 
    gsap.to(circle, { x: x * 30, y: y * 30 });
gsap.to(content,{ x: x * 12, y: y * 10 });
gsap.to(bg,     { x: x * 8,  y: y * 6  });
  • Feel (snappier): 
    gsap.to(circle, { duration: 0.2, ease: "power2.out" });
  • Mobile tolerance: clamp multipliers or reduce them before the tweens.

Pattern 4

Accordion State

Concept: Reveal one item at a time with a reversible open/close.

Live demo

Interaction

Implementation Notes

Core pattern

Give each item its own timeline and only keep one open by reversing the others on click.

View JS for this pattern 
const tl = gsap.timeline({ paused: true, defaults:{ ease: Motion.ease.out, duration: Motion.duration.sm } })
  .fromTo(content, { autoAlpha: 0, maxHeight: 0, y: -6 }, { autoAlpha: 1, maxHeight: 200, y: 0 })
  .fromTo(chevron, { rotate: 0 }, { rotate: 90 }, 0);

item.addEventListener("click", () => {
  const isOpen = tl.progress() > 0 && !tl.reversed();
  timelines.forEach((otherTl, otherItem) => {
    if (otherItem !== item) otherTl.reverse();
  });
  isOpen ? tl.reverse() : tl.play();
});
How to apply
  • Markup: each accordion trigger has data-acc, contains a .accordion__content and .accordion__chevron.
  • Init: import initAccordion and call after DOM ready.
  • Behavior: one-open-at-a-time handled in JS; keep triggers as buttons for accessibility.
  • Example page: FAQ accordion layout
What to tweak
  • Content height: 
    fromTo(content, { maxHeight: 0 }, { maxHeight: 320 })
  • Ease/duration: 
    const tl = gsap.timeline({ defaults:{ duration: 0.2, ease: "power1.out" }});
  • Chevron: 
    fromTo(chevron, { rotate: 0 }, { rotate: 135 })

Implementation Notes

Core pattern

Loop the loader timeline for the loading state, then switch to a completion state before resetting.

View JS for this pattern 
const loopTl = gsap.timeline({ repeat:-1, paused:true, defaults:{ ease:"none" }})
  .fromTo(bar, { xPercent:-100, width:"35%" }, { xPercent:430, duration:1.8 });

const play = () => { label.textContent = "Loading"; gsap.set(bar,{ width:"35%", xPercent:-100 }); loopTl.restart(); };
const complete = () => { loopTl.pause(0); gsap.to(bar,{ xPercent:0, width:"100%", duration: Motion.duration.md, ease: Motion.ease.out }); label.textContent = "Complete"; };
const reset = () => { loopTl.pause(0); gsap.set(bar,{ xPercent:-100, width:"35%" }); label.textContent = "Loading"; };
How to apply
  • Markup: keep the .loader__track and .loader__bar structure.
  • Init: import initLoader and call after DOM ready.
  • Control: wire your own events to play/complete/reset functions or reuse the sample buttons.
What to tweak
  • Bar travel: 
    fromTo(bar, { xPercent:-100, width:"45%" }, { xPercent: 500, duration: 2 })
  • Speed: 
    const loopTl = gsap.timeline({ defaults:{ ease:"none" }})
  .fromTo(bar, {...}, { duration: 1.2 });
  • Visuals: 
    .loader__bar { background: linear-gradient(90deg, #7ad7ff, #f2a3ff); height: 12px; }

const stack = document.querySelector(".toast-stack");
const addToast = () => {

Pattern 9

Tabbed Content Slide

Concept: Move one indicator to show the active view.

Live demo

Interaction

Design tokens

Use consistent spacing, radius, and shadows across patterns.

Motion

Transforms + opacity only; timelines stay reversible.

Handoff

Each pattern is a module with a single init function.

Implementation Notes

Core pattern

Move a single indicator to the active tab and swap the panel state on each click.

View JS for this pattern 
const tabs = gsap.utils.toArray(".tabs__tab");
const panels = gsap.utils.toArray(".tabs__panel");
const underline = document.querySelector(".tabs__underline");
const activate = (name) => {
  const tab = tabs.find((t) => t.dataset.tab === name);
  const panel = panels.find((p) => p.dataset.panel === name);
  tabs.forEach((t) => t.classList.toggle("is-active", t === tab));
  panels.forEach((p) => p.classList.toggle("is-active", p === panel));
  gsap.to(underline, { x: tab.offsetLeft, width: tab.offsetWidth, duration: Motion.duration.sm });
  gsap.fromTo(panel, { autoAlpha: 0, y: 8 }, { autoAlpha: 1, y: 0, duration: Motion.duration.sm });
};
How to apply
  • Keep .tabs__tab buttons with matching data-tab and data-panel values.
  • Import initTabs (via ui.js) and call after DOM ready.
  • Underline slides to the active tab; panels fade/slide into place.
What to tweak
  • Underline speed: 
    gsap.to(underline, { duration: Motion.duration.sm })
  • Panel motion: 
    gsap.fromTo(panel, { y: 8 }, { y: 0 })
  • Tab count: 
    grid-template-columns: repeat(3, 1fr);
